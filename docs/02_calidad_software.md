# Calidad de software

En este documento se seleccionan y justifican los atributos de calidad más relevantes para el SGAB, explicando por qué son importantes y cuáles serían las consecuencias de no atenderlos adecuadamente.

**Seguridad:** Dado que el sistema manipula datos personales y calificaciones, la seguridad es un atributo crítico. Es necesario proteger la confidencialidad e integridad de los datos mediante control de acceso, autenticación y prácticas básicas de seguridad (encriptado de contraseñas, uso de HTTPS, etc...). Si no se prioriza la seguridad, se corre el riesgo de exposición de datos personales, pérdida de integridad en las calificaciones (manipulación) y problemas legales o reputacionales para la institución. Para asegurar la seguridad se pueden definir requisitos no funcionales concretos como autenticación por roles, almacenamiento seguro de contraseñas y registro de auditoría de cambios.

**Mantenibilidad:** El proyecto, aunque pequeño en alcance inicial, debe diseñarse para facilitar futuras modificaciones (correcciones, nuevas funcionalidades). Esto implica una estructura modular lógica, documentación clara y simples contratos de interfaz entre módulos. Si no se piensa en mantenibilidad, las pequeñas mejoras demandarán un esfuerzo desproporcionado y generarán deuda técnica la cual elevará la probabilidad de errores. Acciones concretas para mejorar mantenibilidad incluyen separar responsabilidades (modelo, vista, controlador), documentar decisiones arquitectónicas y mantener pruebas básicas que verifiquen funciones críticas.

**Usabilidad:** Los usuarios suelen tener poco tiempo; si la interfaz no es intuitiva o requiere muchos pasos para tareas comunes, el sistema será rechazado o usado de forma incorrecta. Por eso se deben definir criterios de usabilidad como flujos simples para registrar notas, formularios con validaciones útiles y pantallas de consulta claras. La falta de usabilidad provoca errores de captura, retrabajo y baja adopción. Para mejorarla, podemos usar prototipos básicos (wireframes), validación con un usuario real y requisitos que limiten campos obligatorios innecesarios.
